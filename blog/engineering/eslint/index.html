# eslint

# 快速使用

在安装node的情况下，在项目中：

  1. npm i eslint -D
  2. eslint --init

# 配置

Configuring ESLint

# root

eslint与npm一样，会自动向上查找配置文件。
此配置将eslint的查找配置限定在项目目录中，阻止向父级目录中寻找eslint配置，如eslintrc.*，或package.json。

# extends 继承配置

一个配置文件可以从基础配置中继承已启用的规则。

# 命令

常用命令，配置在package.scripts中:

"scripts": {
  "lint:log": "eslint -c ./.eslintrc --ext .jsx,.js,.vue ./components",
  "lint:fix": "eslint -c ./.eslintrc --fix --ext .jsx,.js,.vue ./components",
}

# eslint-config-prettier

Turns off all rules that are unnecessary or might conflict with Prettier.

This lets you use your favorite shareable config without letting its stylistic choices get in the way when using Prettier.

Note that this config only turns rules off, so it only makes sense using it together with some other config.

{
  "extends": [
    "some-other-config-you-use",
    "prettier"
  ]
}

# eslint-plugin-vue

Official ESLint plugin for Vue.js.

This plugin allows us to check the <template> and <script> of .vue files with ESLint.

Finds syntax errors. Finds the wrong use of Vue.js Directives. Finds the violation for Vue.js Style Guide. ESLint editor integrations are useful to check your code in real-time.
